Here’s a quick way to exercise both the CLI and the (headless) GUI we have in guideXOSCpp.
Build and run
•	Build the C++ project guideXOSCpp (C++14). In Visual Studio: set guideXOSCpp as Startup Project, x64 Debug, Build, then Start Without Debugging. Or use your existing vcxproj.
•	The binary is an interactive console app (server.cpp has main).
CLI smoke test (at the guideXOSCpp prompt)
•	mem — shows allocator usage.
•	alloc 1024 — allocates bytes and prints pointer.
•	tasks — shows scheduler stats.
•	log — prints logger snapshot.
•	spawn worker 3 — starts a demo worker; then plist to list PIDs.
•	send <pid> hello and recv <pid> — demo per-process IPC.
•	bus.sub test 1 — subscribe PID 1 (or your pid) to a channel, bus.pub test hi fanout, then bus.pop test 200.
•	pbytes — shows per-pid memory usage.
•	help — lists commands.
•	exit — quits the server.
Console service test (IPC-based “console”)
•	console.start — starts the console service process (returns pid).
•	console.send hello world — sends a line into the console service.
•	console.pop 500 — pops a reply from console.output (expect “[console] hello world”).
•	console.send exit then console.pop — expect “bye,” and the service exits.
GUI compositor test (headless) Note: The compositor is a stubbed GUI service; there’s no real window drawn. You test via the IPC bus and responses printed by gui.pop.
•	gui.start — starts the compositor (pid logged).
•	gui.win MyWin 320 200 — requests a window. Then:
•	gui.pop — should print GUI: type=1 payload=<id>|MyWin (CreateWindow event).
•	gui.text <id> Hello GUI — replace <id> with the id from the CreateWindow payload, then:
•	gui.pop — should print GUI: type=3 payload=<id>|Hello GUI (DrawText echo).
•	gui.close <id> — request close, then gui.pop — should print GUI: type=2 payload=<id>.
Automating a quick pass
•	You can pipe a script of commands into the exe (e.g., a .txt with lines: mem, console.start, console.send hi, console.pop, gui.start, gui.win Test 200 150, gui.pop, exit).
What to expect
•	All GUI actions are acknowledged via GUI events printed by gui.pop. No OS windows are created yet.
•	The compositor runs until the main process exits (there’s no exposed “bye” for it yet).
